STM32之ld链接脚本详讲

发表于2025-10-29|更新于2025-10-29|MCU开发
|浏览量:

概述

在嵌入式开发中，链接脚本（Linker Script）是一个至关重要的文件，它告诉链接器如何将代码和数据分配到处理器的内存中。对于STM32开发者来说，理解并能够编写自定义链接脚本是必备技能。本文将详细介绍ld格式的链接脚本。

一、链接脚本核心原理：3 大核心模块

STM32 GCC 链接脚本的结构高度固定，核心由ENTRY（入口点）、MEMORY（内存定义）、SECTIONS（段分配） 三部分组成。所有自定义脚本都围绕这三部分展开。

二、逐模块拆解

2.1 第一步：定义程序入口点（ENTRY）

程序上电后首先执行的不是main函数，而是 STM32 启动文件中的Reset_Handler（复位中断服务函数），链接脚本需要通过ENTRY指定这个入口，确保程序从正确的位置开始。

2.2 第二步：定义内存硬件参数（MEMORY）

这是最关键的自定义部分，必须严格匹配你使用的 STM32 型号参数（参数来自芯片数据手册的 “Memory Mapping” 章节）。

2.2.1 内存参数怎么查？

打开 STM32 数据手册，找到 “Memory” 部分，确认两个核心参数：

  • FLASH：起始地址（ORIGIN）、总大小（LENGTH）

    所有 STM32 的 FLASH 起始地址都是0x8000000，差异只在大小。

  • RAM：起始地址（ORIGIN）、总大小（LENGTH）

    所有 STM32 的 RAM 起始地址都是0x20000000，差异同样在大小。

2.2.2 MEMORY 模块写法

格式：内存名 (访问权限) : ORIGIN = 起始地址, LENGTH = 大小

  • 访问权限：rx（FLASH 只读可执行）、xrw（RAM 可读可写可执行）。

2.3 第三步：配置堆和栈大小（Heap & Stack）

堆和栈是 RAM 中的 “临时存储区”，大小需要根据应用需求调整，默认值仅为基础参考。

2.3.1 堆（Heap）

  • 用途：用于malloc、free等动态内存分配。
  • 默认值：_Min_Heap_Size = 0x200（512 字节）。
  • 调整原则：如果程序用了大量动态内存（比如链表、缓冲区），需要增大（如0x1000即 4KB）；如果不用动态内存，可设为0x0。

2.3.2 栈（Stack）

  • 用途：用于函数调用、局部变量存储、中断现场保护。
  • 默认值：_Min_Stack_Size = 0x400（1024 字节）。
  • 调整原则：函数嵌套深、局部变量大（如大数组）时，需要增大（如0x800即 2KB）；栈溢出会导致程序无征兆跑飞，这是最常见的问题之一。

2.4 第四步：分配程序段（SECTIONS）

这是脚本最复杂的部分，但大部分的内容无需修改，只需理解各段的作用，确保与硬件匹配。核心是将编译器生成的 “段”（如代码段、数据段）映射到前面定义的 FLASH 或 RAM 中。

2.4.1 核心段解析

（1）存放在 FLASH 中的段

这些段是 “只读 / 可执行” 的，程序下载时会被烧录到 FLASH。

段名 用途 关键配置
.isr_vector 中断向量表（复位、中断服务函数地址） 必须放在 FLASH 起始地址，用KEEP(*(.isr_vector))防止被编译器优化删除。
.text 程序代码（main函数、普通函数等） 包含.text、.text*（所有代码相关段），用ALIGN(4)保证 4 字节对齐（Cortex-M 要求）。
.rodata 只读数据（如const int a = 10;、字符串常量） 同样 4 字节对齐，避免访问错误。
.init_array/.fini_array 程序初始化 / 退出时执行的函数数组（如全局对象构造 / 析构） 用KEEP保留，确保初始化函数不被优化。
.glue_7/.glue_7t ARM 与 Thumb 指令兼容段 必须保留，因 STM32 的 Cortex-M 内核用 Thumb-2 指令集，可能混合 ARM/Thumb 指令，需此段实现指令过渡，删除会导致指令执行错误。

（2）存放在 RAM 中的段

这些段是 “可读可写” 的，程序运行时会加载到 RAM。

段名 用途 关键配置
.data 初始化的全局变量（如int a = 10;） 数据会先存放在 FLASH，上电后由启动文件复制到 RAM（>RAM AT>FLASH）。
.bss 未初始化的全局变量（如int a;） 上电后由启动文件清零，不占用 FLASH 空间（仅占 RAM）。
._user_heap_stack 堆和栈的存储区，用于检查 RAM 是否足够 由堆大小 + 栈大小组成，用ALIGN(8)保证 8 字节对齐。
.RamFunc 需在 RAM 中执行的函数（如快速中断函数、高频调用函数） \1. 脚本中保留*(.RamFunc)；2. 代码中用属性声明函数：__attribute__((section(".RamFunc"), noinline)) void fast_isr(void) {}

总结

编写 STM32 GCC 链接脚本的核心不是 “从零写”，而是 “按需改”——固定结构不变，仅修改 3 处关键参数：

  1. MEMORY模块的 FLASH/RAM 大小；
  2. 堆和栈的大小；
  3. 确认.isr_vector、.data、.bss段的映射关系正确（无特殊要求不需要更改）。
